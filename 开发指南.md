## 项目概述

### 设计理念

LangChainSkills 是一个基于 LangChain 的可扩展智能体系统，采用 **Skills（技能）动态加载** 架构。核心设计理念：

- **按需加载**：Agent 启动时只加载 Loader Tools，实际工具在需要时才激活
- **动态过滤**：通过中间件机制实现运行时工具过滤，减少 token 使用和错误率
- **模块化设计**：每个 Skill 是独立的模块，支持热插拔

### 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                         Agent                                │
│  ┌─────────────────────────────────────────────────────┐   │
```

### 工作流程

1. **初始化阶段**：
   - 加载配置文件
   - 创建 SkillRegistry
   - 自动发现并加载 Skills
   - 注册所有工具到 Agent

2. **运行阶段**：
   - Agent 接收用户输入
   - SkillMiddleware 拦截请求
   - 根据 `skills_loaded` 状态过滤工具
   - 模型只看到相关工具

3. **Skill 激活**：
   - 模型调用 Loader Tool
   - Loader Tool 返回 `Command` 更新状态
   - `skills_loaded` 添加新 Skill 名称
   - 下次请求时该 Skill 的工具可见

## 核心概念

### Skill（技能）

Skill 是系统的核心抽象，每个 Skill 代表一种能力单元。

**组成部分**：

| 组件 | 说明 | 可见性 |
|------|------|--------|
| Loader Tool | 激活 Skill 的入口工具 | 始终可见 |
| Tools | Skill 的实际功能工具 | 激活后可见 |
| Instructions | Skill 使用说明 | 激活时返回给 Agent |
| Metadata | Skill 元数据（名称、版本、标签等） | 注册时使用 |

### SkillRegistry（注册中心）

负责 Skill 的生命周期管理：

```python
registry = SkillRegistry()

# 自动发现加载
registry.discover_and_load(skills_dir=Path("./skills"))

# 手动注册
registry.register(my_skill)

# 获取工具
all_tools = registry.get_all_tools()
loader_tools = registry.get_all_loader_tools()
filtered_tools = registry.get_tools_for_skills(["data_analysis"])
```

### SkillMiddleware（中间件）

实现动态工具过滤的核心组件：

```python
skill_middleware = SkillMiddleware(
    skill_registry=registry,
    verbose=True
)
```

**工作原理**：

1. 拦截每次 `model.call()` 请求
2. 从 `request.state` 获取 `skills_loaded`
3. 调用 Registry 获取对应工具
4. 使用 `request.override(tools=filtered_tools)` 替换工具列表

### SkillState（状态管理）

三种状态管理模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| Replace | 每次激活替换之前的状态 | 单任务场景 |
| Accumulative | 累积所有激活的 Skills | 多任务协同 |
| FIFO | 最多保持 N 个 Skill | 资源受限场景 |

```python
# 替换模式
class SkillState(MessagesState):
    skills_loaded: Annotated[List[str], skill_list_reducer] = []

# 累积模式
class SkillStateAccumulative(MessagesState):
    skills_loaded: Annotated[List[str], skill_list_accumulator] = []

# FIFO 模式（最多 3 个）
class SkillStateFIFO(MessagesState):
    skills_loaded: Annotated[List[str], skill_list_fifo(3)] = []
```

### 目录结构

### 通用技能 (Common Skill)

通用技能是纯 Instructions 技能，无实际工具，适用于提供文档查询指南、最佳实践等。

**文件格式**（`skills/common/xxx.md`）：

```markdown
---
name: skill-name
description: Skill 功能描述
---

# Skill Title

## Overview
...

## Instructions
...
```

**特点**：
- 使用 YAML front matter 定义 `name` 和 `description`
- 正文是详细的 Instructions 指导
- 无需 `skill.py` 文件
- 自动被 Registry 加载

#### 1. 创建 Skill 目录


#### 2. 编写 skill.py

```python
"""
My Skill - 自定义技能说明
"""

from pathlib import Path
from typing import List
from langchain_core.tools import tool, BaseTool
from langchain_core.messages import ToolMessage
from langgraph.types import Command

from core.base_skill import BaseSkill, SkillMetadata


class MySkill(BaseSkill):
    """自定义 Skill 实现"""

    @property
    def metadata(self) -> SkillMetadata:
        return SkillMetadata(
            name="my_skill",                      # 唯一标识
            description="我的自定义技能描述",      # 功能说明
            version="1.0.0",                      # 版本号
            tags=["custom", "demo"],              # 标签
            dependencies=[],                       # 依赖库
            author="Your Name"                    # 作者
        )

    def get_loader_tool(self) -> BaseTool:
        """返回 Loader Tool"""
        skill_instance = self

        @tool
        def skill_my_skill(runtime) -> Command:
            """
            Load my_skill capabilities.
            
            Call this tool when you need to:
            - 功能说明 1
            - 功能说明 2
            """
            instructions = skill_instance.get_instructions()

            return Command(
                update={
                    "messages": [ToolMessage(
                        content=instructions,
                        tool_call_id=runtime.tool_call_id
                    )],
                    "skills_loaded": ["my_skill"]  # 更新状态
                }
            )

        return skill_my_skill

    def get_tools(self) -> List[BaseTool]:
        """返回实际工具列表"""
        return [
            self._create_tool_1(),
            self._create_tool_2(),
        ]

    def _create_tool_1(self) -> BaseTool:
        """创建工具 1"""
        @tool
        def tool_function(param1: str, param2: int = 10) -> str:
            """
            工具功能描述。
            
            Args:
                param1: 参数 1 说明
                param2: 参数 2 说明
            
            Returns:
                返回结果说明
            """
            try:
                # 实现逻辑
                result = f"处理结果: {param1} - {param2}"
                return result
            except Exception as e:
                return f"Error: {str(e)}"

        return tool_function

    def _create_tool_2(self) -> BaseTool:
        """创建工具 2"""
        # ... 类似实现
        pass


def create_skill(skill_dir: Path) -> BaseSkill:
    """
    工厂函数 - 必须定义
    
    Registry 自动加载时会调用此函数
    """
    return MySkill(skill_dir)
```

#### 1. 工具设计原则

- **单一职责**：每个工具只做一件事
- **明确描述**：docstring 要清晰说明功能和参数
- **错误处理**：捕获异常并返回友好提示
- **依赖检查**：动态导入依赖库，给出安装提示

#### 2. Loader Tool 设计

- 名称格式：`skill_<skill_name>`
- 返回类型：`Command` 对象
- 状态更新：`skills_loaded` 列表
